# Система управления бронированиями

Это REST API приложение для управления бронированиями, разработанное на Java с использованием Spring Boot. Приложение позволяет создавать, обновлять, отменять и подтверждать бронирования, а также искать их по различным фильтрам.

## Стек технологий

- **Java 17**
- **Spring Boot 3**
- **Spring Data JPA** (Hibernate)
- **Spring Web MVC**
- **Jakarta Validation**
- **PostgreSQL** 
- **Maven** (для сборки)

## Основные возможности

- Создание нового бронирования
- Получение бронирования по ID
- Получение списка бронирований с фильтрацией и пагинацией
- Обновление существующего бронирования
- Отмена бронирования
- Подтверждение бронирования с проверкой доступности
- Валидация входных данных
- Обработка конфликтов бронирований (пересечение дат)

## API Endpoints

| Метод   | URL                      | Описание |
|---------|--------------------------|----------|
| GET     | `/{id}`                  | Получить бронирование по ID |
| GET     | `/`                      | Получить список бронирований с фильтрацией (roomId, userId, pageSize, pageNumber) |
| POST    | `/`                      | Создать новое бронирование |
| PUT     | `/{id}`                  | Обновить существующее бронирование |
| DELETE  | `/{id}/cancel`           | Отменить бронирование |
| POST    | `/{id}`                  | Подтвердить бронирование |

## Логика работы

1. **Создание бронирования**
    - Проверяется, что статус не передан (автоматически устанавливается `PENDING`)
    - Выполняется проверка: дата окончания должна быть позже даты начала
    - Бронирование сохраняется в БД

2. **Обновление бронирования**
    - Проверяется, что бронирование существует
    - Обновление возможно только для бронирований со статусом `PENDING`
    - Выполняется проверка корректности дат

3. **Отмена бронирования**
    - Отменить можно бронирование со статусом `PENDING`
    - Статус меняется на `CANCELLED`

4. **Подтверждение бронирования**
    - Подтвердить можно только бронирование со статусом `PENDING`
    - Выполняется проверка на пересечение с другими подтверждёнными бронированиями
    - При отсутствии конфликтов статус меняется на `APPROVED`

5. **Поиск бронирований**
    - Поддерживается фильтрация по `roomId` и `userId`
    - Реализована пагинация (по умолчанию: 10 элементов, страница 0)

## Обработка ошибок

Приложение выбрасывает исключения в следующих случаях:
- `EntityNotFoundException` – если бронирование не найдено
- `IllegalArgumentException` – при нарушении валидации дат или статуса
- `IllegalStateException` – при попытке изменить/подтвердить бронирование с недопустимым статусом, либо при обнаружении конфликта дат

## Запуск приложения

### Требования
- Java 17+
- Maven 3.6+
- PostgreSQL (или другая БД)
